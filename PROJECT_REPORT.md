## Project Overview

Social Media App is a full-stack web application that enables users to create, view, and delete short text posts. The platform is designed to promote concise expression while maintaining simplicity and performance. Users can view all posts publicly without authentication, while only authenticated users can create and manage posts.

This project follows a client–server architecture, with a React (Vite) frontend and an Express.js backend, connected through REST APIs and backed by MongoDB for persistent storage.

## Deployment Report

### Netlify (Frontend)

**Why Netlify?**
- Excellent support for static frontend frameworks (React/Vite)
- Automatic builds on Git commits
- CDN-based global delivery
- Easy environment variable management
- Free SSL/HTTPS
- Seamless Git integration

**Frontend Deployment Steps**

1. **Repository Structure**
   ```
   /frontend
   ├── package.json
   ├── vite.config.js
   ├── src/
   ├── public/
   └── dist/ (generated by build)
   ```

2. **Netlify Build Configuration**
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `dist`

3. **Environment Variables (Set in Netlify UI)**
   ```
   VITE_API_URL = https://<your-backend-domain>/api
   ```

4. **Routing Configuration**
   Created `frontend/public/_redirects` file:
   ```
   /* /index.html 200
   ```
   This ensures React Router handles all routes correctly.

5. **Live URL:** https://dreamy-stroopwafel-f44aff.netlify.app/

### Backend Deployment

**Recommended Platforms:** Render, Heroku, Railway, Fly.io

**Backend Configuration**
- Express server listens on `process.env.PORT`
- Production-ready middleware (CORS, JSON parsing, error handling)
- MongoDB connection via environment variables
- JWT secret stored securely in env variables

**Environment Variables (Required)**
```
PORT = 5000
MONGO_URI = mongodb+srv://username:password@cluster.mongodb.net/socialmedia
JWT_SECRET = <secure-random-secret>
```

**Example Setup (Render/Similar)**
- **Runtime:** Node.js
- **Root directory:** `backend`
- **Build command:** `npm install`
- **Start command:** `npm start`

## API Endpoints

### Authentication
- `POST /api/auth/signup` - Register new user
- `POST /api/auth/login` - Login user
- `GET /api/auth/verify` - Verify JWT token

### Posts
- `GET /api/posts` - Fetch all posts (public)
- `POST /api/posts` - Create new post (authenticated)
- `DELETE /api/posts/:id` - Delete own post (authenticated)

## Troubleshooting

**Issue:** Login/Signup fails with CORS error
- **Solution:** Ensure backend CORS allows frontend domain and API URL is correct in `VITE_API_URL`

**Issue:** Posts not loading
- **Solution:** Check MongoDB connection and API endpoint in network tab

**Issue:** Token expiration errors
- **Solution:** Clear localStorage and login again; check JWT_SECRET consistency
